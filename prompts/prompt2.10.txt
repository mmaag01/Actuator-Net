# Context

Hold-regime bias contributes ~46% of test squared error but training data lacks sustained holds, so we cannot fix it with more data. The remaining lever is to change what the network is asked to learn. `torEst` (EPOS4 internal torque estimate, iq-derived) has RMSE 19.8 Nm against `torAct` (Test 1) — it's a bad standalone predictor but a reasonable baseline. If we make the network learn the RESIDUAL `torAct - torEst` instead of `torAct` directly, the target variance drops substantially, and the network's capacity can be spent on the part of the signal that `torEst` doesn't explain.

This is essentially a skip connection from `torEst` to the output: `pred = torEst + f_theta(features)` where f_theta is the NN. At inference, predicted torque = NN output + current torEst value. Inspired by residual networks and by the hybrid-model narrative in Lee et al. (HWANG_really_accurate.pdf) where a physics baseline carries the dominant signal and a learned term corrects it.

# Task

   - During training, loss is MSE on the normalized residual.
   - During evaluation, `pred_torque = scaler_y.inverse_transform(model_output) + torEst[t]`. Report metrics (RMSE, MAE, regime breakdown, error at idx=38180) on the RECONSTRUCTED torque, not the residual — this must be apples-to-apples with the v1 results.

# Interpretation block

- "Residual framing helps broadly" if overall RMSE drops >10% on both models AND hold RMSE drops >20% — strongest possible outcome
- "Residual framing fixes holds only" if hold RMSE drops >30% but overall RMSE moves <5%
- "Residual framing fixes ringing only" if idx=38180 error drops >50% but aggregate metrics barely move  
- "Residual framing neutral" if everything moves <5% — would indicate the NN was already effectively learning the residual implicitly (consistent with architecture convergence being a fundamental limit, not a choice)
- "Residual framing hurt" if metrics degrade — means the normalization statistics for the residual space are poorly matched to torEst's error distribution

# Requirements

- Record the target residual statistics (mean, std, skew, kurtosis) on the training set so we know what the network was actually optimizing. Print these at the start of training.
- Inference reconstruction must be tested with an assertion: run the v1 test set through the residual pipeline using torEst as the skip signal and confirm output shape and magnitude are reasonable before declaring metrics.

# Out of scope

- Combining residual learning with accelAct smoothing, ablations, or longer SEQ_LEN. Those combinations come after this single-variable experiment shows whether residual framing helps.
- Any other skip-connection design (e.g. skip from `i` or `torDes`). torEst is the highest-signal candidate; start there.
